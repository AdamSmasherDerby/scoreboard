Notes on Browsers

There are pros and cons of using the various browsers that are out there.

  • Firefox

    Firefox 4 does very well and is available pretty much anywhere. Firefox 3 is not fully compatible with the HTML5 specifications that the scoreboard uses, so the scoreboard will not look correct; at least version 4 of Firefix should be used. One notable downside to Firefox is its "fullscreen" mode (toggle via F11) is not really full screen - it leaves a small white border at the top of the screen. Also, Firefox has a annoying thing called "Offline mode" that will (incorrectly) prevent you from accessing *any* page, including the scoreboard server running on your local machine. You have to disable offline mode for it to work.

  • Google Chrome

    Google Chrome is good and available on most platforms. It is better than Firefox at fullscreen, as it actually does do fullscreen (no strange white bar at the top). However, it's got its own problems; it is not quite as stable as Firefox, and it does not handle "offline" correctly in Linux.

  • Safari

    Apple Safari is a good browser, however it is slightly lagging in correct support for some of the newer HTML5 features that the scoreboard uses, such as full/correct support for the css3 "border-radius" property. So currently, there are better alternatives such as Firefox 4 or Chrome. In the future, I expect Safari to fully/correctly implement the HTML5 specification and be usable with the scoreboard.

  • Internet Explorer

    Internet Explorer has a long history of completely disregarding internet standards, but it's getting slightly better. However, it's the only browser that lacks a key feature that the scoreboard is fundamentally based on, so it simply will not work. For technically minded people, the specific problem is that IE does not allow "expando" properties on XML document elements; this prevents the jQuery data() function from working on XML document elements, which is an essential part of how the scoreboard currently works.

    So, you cannot use IE with the scoreboard.

Using the ScoreBoard with no internet (or local) connection

You don't need an internet connection to use the scoreboard; since you're running it on your own machine, the web browsers should connect to "localhost" (unless you're using multiple computers and have set up your own private network). However, if you are running with no active network interface, Firefox goes into "offline mode" and will not let you connect, even to localhost. You have to click "File" then uncheck "Offline mode". Note that Firefox will automatically re-enter "offline mode" if any of your network interfaces change state, for example if you plug in a network cable and then disconnect it. Also, Google Chrome on Linux, when it detects you have no active network interfaces, simply won't let you connect to anything, and there's nothing you can do about it. There is a bug open:
http://code.google.com/p/chromium/issues/detail?id=41408

Image Sizes

The scoreboard currently uses 3 different types of images, each of which should be a specific size (or aspect ratio) for best viewability. Note that team logo and ad images do not have to be specifically these sizes, but otherwise they will not completely fit into the space provided in the scoreboard display, there will be some "blank" space above or beside them, depending on their specific aspect ratio. The fullscreen images are adjusted/stretched to fit across the entire 4:3 scoreboard.

  • Team Logo images should be 640x240, or aspect ratio 2.667:1
  • Fullscreen images should be 1600x1200, or aspect ratio 4:3 (the main scoreboard is fixed at 4:3 ratio and resizes to fit the browser window)
  • Ad images should be 1600x320, or aspect ratio 5:1

Configuration files

The main configuration file is the crg.scoreboard.properties Java Properties file. It is located in the config directory. This file controls many core settings for the scoreboard, but you should not need to change anything in this file.

The scoreboard loads its initial data from xml file(s) in the config directory. First, it will try to load the first valid auto-saved scoreboard from the config/autosave directory; if there are no (valid) auto-saved files there, it will load (in alphabetical order) all the xml files from the config/default directory.
